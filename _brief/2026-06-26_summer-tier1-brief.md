## 2. 修正後 Tier 1: 網走市補助金活用ガイド実装

### 2-1. 何を作るか

`abashiri/index.html` 内に **「網走市民の方へ：エアコン設置工事費助成制度の活用ガイド」セクション**を新規追加する。

### 2-2. なぜ作るか（戦略的背景）

1. **競合ゼロの独自コンテンツ**: 網走市の SERP 上位（くらしのマーケット / 家仲間コム / ミツモア / Zehitomo 等）は全て全国系プラットフォームで、補助金情報を持っていない。ac-kitami が補助金ガイドを実装すれば、即座に「網走市 エアコン 補助金」「網走市 エアコン 助成」で **競合不在の上位獲得が見込める**
2. **問い合わせ動機の強化**: 補助金活用は顧客側に「申請書類の準備・受領証の取得」等の支援ニーズがある。「補助金活用サポート込み」を訴求すれば、価格比較ではなく **付加価値で選ばれる立ち位置** を作れる
3. **AI 検索引用源**: 補助金制度の具体情報を構造化して載せれば、ChatGPT / Perplexity / Google AI Overview から **引用元として参照されやすい**（口コミ・評判系の SERP がプラットフォーム独占でも、AI 検索経由で露出を取れる）

### 2-3. どう作るか（推奨実装方針）

#### A. コンテンツ構造

```
<section> 網走市民の方へ：エアコン設置工事費助成制度の活用ガイド
  <h2> 網走市のエアコン設置工事費助成制度とは
    - 制度の概要（公式情報を要約・出典明示）
    - 対象者・対象工事・助成額（必ず網走市公式サイトで最新確認）
    - 申請の流れ（書類準備 → 工事 → 受領証 → 申請）

  <h3> 当店が提供する「補助金活用サポート」
    - 申請書類に必要な「工事費明細書・領収書」を制度要件に沿って発行
    - 工事日程・申請期限の調整サポート
    - 受領証の発行

  <h3> 補助金活用のよくある質問（FAQ）
    Q1: 申請はいつまで？ → A1（最新の網走市公式情報を引用）
    Q2: どの工事が対象？ → A2
    Q3: 申請手続きは自分でやる？ → A3（当店がサポートする範囲を明示）

  <CTA>
    「網走市の補助金活用についてお問い合わせ」
    電話 + LINE + フォーム
</section>
```

#### B. SEO 実装要件

- **新規セクション内に H2「網走市のエアコン設置工事費助成制度」**
- 共起語: 助成 / 補助金 / 申請 / 受領証 / 工事費明細 / 網走市 / 申請期限 / 対象工事 / 対象者
- 構造化データ: `abashiri/index.html` の既存 FAQPage JSON-LD に **補助金 FAQ を追加**
- アンカーリンク: トップページから `/abashiri/#josei` 等で網走補助金セクションへ直接遷移できるよう FV に小バッジ追加検討（網走市民向け訴求）

#### C. SEO 死守ライン（厳守）

- ✅ **既存の title / meta description / og:* / canonical / H1 は触らない**
- ✅ 既存 LocalBusiness / BreadcrumbList JSON-LD は変更しない
- ✅ 既存 FAQPage JSON-LD は **追加のみ**（既存 Q&A は触らない）
- ✅ 既存 H2 / H3 の順序・文言は変更しない（新規 H2 は最後に追加 or 適切な位置に挿入）
- ✅ 既存画像 URL / alt は触らない

#### D. 一次情報の取り方（厳守）

- 網走市公式サイトの当該ページを **必ず一次ソースとして参照** し、URL を構造化データ・本文中に明示
- 助成額や期限などの数値は **必ず公式サイトで最新確認**（古い情報の掲載は信用毀損 + 法令リスク）
- 一次ソース URL を「最終確認日」と併記（例: 「2026-06-26 時点・網走市公式サイトより」）

## 3. Tier 1 補強（時間が余ったら）

### 3-1. 「最短翌日対応」FAQ ブロック化

トップに既に訴求はあるが、**FAQ 形式で再構造化**することで AI 検索引用率を上げる:

```
Q: エアコンの取付は何日くらいで来てもらえますか？
A: 北見市内は最短翌日対応が可能です。在庫品の場合は即日対応も承ります。
   出張費は北見市内無料、美幌町まで無料、その他地域は要相談です。
```

実装場所: トップの FAQ セクション（既存）に Q&A を **追加のみ**

### 3-2. 料金表セクションへのトップアンカー追加

既存料金セクションへのスムーズな遷移を作るため、FV 下に小さく「料金を見る」アンカーボタンを追加検討（CSS 変更のみ・既存 HTML 構造を壊さない）
