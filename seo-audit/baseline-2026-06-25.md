### 所感

- **PC は既に十分速い** (全ページ 88 点以上・LCP 2 秒以下)。改修対象外にしてよい
- **モバイルが全ページ問題** (56-61 点・LCP 7-10 秒)。改修の主戦場
- works/001 が最も改善余地大（LCP 10.1 秒）
- TBT は全ページ 0 ms（ブロッキング時間なし・JS は問題なし）
- CLS は全ページ 0.03 以下（レイアウトシフトなし・既に Web Vitals 緑域）
- **真の課題は LCP/FCP**。すなわち「最初のコンテンツ表示が遅すぎる」点だけ。Phase 1+2 のレンダーブロック解消（フォント非同期 + キャッシュ強化）が直接効く構造

### 主要な改善余地（PageSpeed Insights の「改善できる項目」）

- [ ] 使用していない CSS の削減（89 KiB）→ icons.css と tailwind.css が主因

- [ ] 次世代フォーマットでの画像の配信 → FV 画像 WebP 化未実施
