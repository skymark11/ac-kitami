# 施工事例ページ量産スキル

## このスキルの狙い

施工事例のページ作成が止まっていた理由は、**写真を1枚ずつ「before-01」「process-02」と手動リネームする手間**でした。
このスキルでは、その手間をゼロにします。ユーザーは**スマホで撮った写真をフォルダにそのままドサッと入れる**だけ。あとはClaudeが全部やります。

---

## ユーザーがやること（3ステップ）

### Step 1: 写真を raw/ フォルダに入れる

```
images/works/{area}/{NNN}/raw/
├── IMG_1234.jpg    （スマホで撮ったまま）
├── IMG_1235.heic
├── IMG_1236.jpg
└── ...
```

- 場所: `images/works/{area}/{NNN}/raw/` （例: `images/works/abashiri/003/raw/`）
- **リネーム不要**
- **形式混在OK**（JPEG / HEIC / PNG / WebP）
- **順番もランダムでOK**（撮影日時で自動ソートされる）
- **アンケート写真も一緒に入れてOK**

### Step 2: info.txt を書く

`images/works/{area}/{NNN}/info.txt` に、`info.txt.example` を参考にして書く。

最低限必須: 施工日・地域・建物・工事内容・機種
任意: 費用・工夫した点・お客様コメント

### Step 3: Claude に依頼

「網走003 の施工事例ページお願い」と言う。

### Step 4: 担当者ヒアリングに答える（必須・最重要）

Claude が「担当者ヒアリング」として以下を質問してきます。
**この内容こそが独自コンテンツの核**になり、SEOにもユーザー信頼にも直結する最重要パートです。

質問項目:

1. **どの作業が一番大変でしたか?**（物理的・技術的に難しかったこと）
2. **どんな工夫・苦労がありましたか?**（標準では対応できず独自の方法を取った場面）
3. **結果的に良かった点は?**（完成後の手応え、見映え、お客様の反応）
4. **お客様の状況や反応は?**（困りごとがどう解決したか、コメント）
5. **同様の工事を検討している方へのアドバイス**

口頭で話した内容をテキストにしてClaudeに送ってください（スマホの音声入力でOK）。
情報が少なくても、Claude が「もう少し詳しく」と聞き返します。

事前に info.txt の「担当者ヒアリング」欄に書いておけば、Step 4 はスキップできます。

## Claude がやること（フェーズ）

### Phase 0: 環境準備

1. このスキル（`SKILL.md`）を全部読む
2. `unified-header-footer` スキルも読む
3. 同じフォルダの以下ファイルを確認:
   - `prepare-photos.py` - 写真前処理スクリプト
   - `detail-page-template.html` - HTMLテンプレ
   - `info.txt.example` - info.txt 例
4. `info.txt` の中身を読む
5. 同じ area の既存施工事例（`/{area}/works/`配下）を一覧確認

### Phase 1: 写真の前処理（スクリプト1発）

```bash
python .agents/skills/works-case-builder/prepare-photos.py images/works/{area}/{NNN}/raw/
```

このスクリプトが自動でやること:
- raw/ 内の全画像を読み込み（JPEG / HEIC / PNG / WebP 対応）
- **EXIFの撮影日時で時系列ソート**（無いファイルは更新日時）
- WebP変換、800px幅にリサイズ、品質85で統一
- 親ディレクトリに `photo-001.webp`, `photo-002.webp`, ... の連番で出力
- 縦写真は縦のまま、横写真は横のまま（アスペクト比保持）

出力後、raw/ の元ファイルはそのまま残る（バックアップ）。

### Phase 1.5: 担当者ヒアリング（必須・最重要）

**このフェーズはスキップ厳禁。** ページの独自性とSEO評価を決定づける核心です。

#### 判断分岐

- `info.txt` の「担当者ヒアリング」欄に既に内容があれば → 読み込んで Phase 2 へ
- 書かれていない場合 → 以下の質問をユーザーに投げる:

```
施工事例ページに「担当者の声」として掲載したいので、口頭でも箇条書きでもいいので教えてください。

1. どの作業が一番大変でしたか?
   （物理的・技術的に難しかったこと）

2. どんな工夫・苦労がありましたか?
   （標準では対応できず独自の方法を取った場面）

3. 結果的に良かった点は?
   （完成後の手応え、見映え）

4. お客様の状況や反応は?
   （困りごとへの解決、コメントなど）

5. 同様の工事を検討している方へのアドバイス
   （これから検討する方へ伝えたいこと）

スマホの音声入力でOK。短い文でも構いません。
```

#### 回答を受け取った後

1. 回答が短い・曖昧な場合 → 「もう少し詳しく聞かせてください。例えば〜」と1〜2点だけ深掘り
2. 内容を整理して以下3つに振り分け:

| 振り分け先 | 内容 | 表現 |
|---|---|---|
| **施工のポイント** | 技術的に重要な工夫（番号付きリスト4-5項目） | 「〇〇を使用」「△△に配慮」 |
| **担当者より** | 人間味のある語り、感想 | 「今回の施工は〜」「お客様にも〜」 |
| **お客様の声** | お客様アンケート or 担当者経由の声 | 引用ブロック |

3. 内容をそのままコピペしない。**Claudeが文体を整える**:
   - 話し言葉 → 読みやすい文章
   - 専門用語に簡単な補足を入れる（例: 「コア抜き（穴あけ作業）」）
   - 過度なへりくだりは抑える、ただし職人らしさは残す
   - 1段落あたり3-4文程度に整形

4. 整形後、ユーザーに確認してもらう:
   ```
   担当者の声を以下のようにまとめました。
   ニュアンスや事実関係に違いがあれば教えてください。
   
   ## 施工のポイント
   ...
   
   ## 担当者より
   ...
   ```

5. OK が出たら info.txt にも追記保存（次回参考用）

### Phase 2: 内容判別と分類

`photo-001.webp` から順番に **Read tool で全部読む**（4-8枚ずつ並列で）。

各写真を以下のテーブルに当てはめる:

#### Before カテゴリ
| ラベル | 写真の特徴 |
|---|---|
| 外観 | 建物全体、エアコンが付いていない |
| 室内設置場所 | 壁・天井（設置予定箇所） |
| 分電盤・既存設備 | ブレーカー、配電盤、コンセント |
| 採寸・現調 | メジャー、寸法確認 |

#### Process カテゴリ
| ラベル | 写真の特徴 |
|---|---|
| 搬入・梱包 | エアコン本体の箱、開封、型番ラベル |
| 採寸・墨出し | 外壁採寸（架台用） |
| 穴あけ | コア抜き、ドリル作業 |
| 背板取付 | 室内機の据付板を壁に固定 |
| 室内機本体取付 | エアコン本体を背板に設置中 |
| 配管接続 | 銅管・電線の接続 |
| 室外機設置 | 架台への据置、固定 |
| 動作確認 | 試運転、職人が確認中 |
| **お客様アンケート** | 記入済み用紙（重要：別途活用） |

#### After カテゴリ
| ラベル | 写真の特徴 |
|---|---|
| 外観全景 | 建物全体、エアコン1台付き |
| 外観ズーム | 室外機+配管+室内機部分のクローズアップ |
| 室内機 | 壁掛けエアコン本体（正面） |
| 室外機 | 据置完了 |
| 専用コンセント | 増設したコンセント（屋内） |
| 化粧モール | 配線カバー（屋内） |
| ドレン配管 | 排水パイプ処理 |

判別結果を以下のような対応表として整理:

```
photo-001.webp = Before/外観 (アパート2階全体、エアコン未設置)
photo-002.webp = Before/分電盤 (古い配電盤)
photo-003.webp = Process/搬入 (IRA-2206RZ 梱包箱)
...
photo-040.webp = Process/お客様アンケート
```

### Phase 3: 物理リネーム

判別結果に基づいて、photo-NNN.webp を before/process/after の連番にリネーム。

Bash例:
```bash
mv photo-001.webp before-01.webp   # 外観
mv photo-002.webp before-02.webp   # 分電盤
mv photo-003.webp process-01.webp  # 搬入
...
mv photo-040.webp process-99.webp  # アンケート（最後にまとめる）
```

リネーム後の構造:
```
images/works/{area}/{NNN}/
├── before-01.webp  ~  before-NN.webp
├── process-01.webp ~  process-NN.webp
├── after-01.webp   ~  after-NN.webp
├── info.txt
└── raw/  (元ファイル、参照用に残す)
```

### Phase 4: Before/Afterペア選定（FV用）

**優先順位（強い順）:**

1. **外観のペア（最強）**
   - Before: 「外観・エアコン未設置」
   - After: 「外観全景・エアコン設置済み」
   - 同じアングル・同じ建物面 が理想

2. 室内設置場所のペア
3. 機能要素のペア（分電盤 → 専用コンセント）

**禁じ手:**
- 同じ場所じゃないペア（Before=分電盤 + After=外観 など）
- 縦写真と横写真をそのまま横並び（CSS `aspect-ratio:1/1;object-fit:cover;` で揃える）

### Phase 5: 施工工程の選定（4枚、時系列）

| 順番 | 推奨カテゴリ |
|---|---|
| 1 | 搬入・梱包 OR 採寸・現調 |
| 2 | 穴あけ OR 背板取付 |
| 3 | 室内機本体取付 |
| 4 | 室外機設置 OR 配管接続 |

該当ない工程はスキップ。4枚に満たなければ3枚でもOK。

**キャプションは写真の中身に正直に書く**:
- 「配管穴あけ」と書くなら実際の穴あけ写真でないとダメ
- 該当ないなら「エアコン搬入」など実際の中身に合わせる

### Phase 6: 完成写真の選定（4枚推奨）

完成物を多角度で:
1. 室内機（壁掛けエアコン本体）
2. 専用コンセント・分電盤
3. 化粧モール配線
4. 室外機・配管立ち下げ

**4枚（PC: 2x2、SP: 2x2）が視覚的バランス最良**。6枚は重い、3枚でもOK。

### Phase 7: HTML生成

`detail-page-template.html` をベースに、以下を置換:

| 変数 | 例 |
|---|---|
| `{{AREA_PATH}}` | `abashiri` |
| `{{AREA_NAME}}` | `網走市` |
| `{{CASE_NUMBER}}` | `001` |
| `{{TITLE}}` | `網走市潮見でアパートにエアコン新規設置｜コンセント増設工事も対応` |
| `{{DESCRIPTION}}` | 100-120文字、行動喚起含む |
| `{{H1}}` | TITLE と同等またはやや短く |
| `{{LEAD}}` | タイトル下のリード文1-2行 |
| `{{BEFORE_IMG}}` | `before-NN.webp` |
| `{{AFTER_IMG}}` | `after-NN.webp` |
| `{{DATE}}` | `2025年8月3日` |
| `{{LOCATION}}` | `北海道網走市潮見6丁目` |
| `{{BUILDING}}` | `アパート（2階）` |
| `{{WORK_DESC}}` | `エアコン新規設置 + 専用コンセント増設` |
| `{{MODEL}}` | `アイリスオーヤマ IRA-2206R（6畳用）` |
| `{{COST}}` | `209,220円（税込）` |
| `{{POINTS}}` | 施工のポイント（番号付きHTMLリスト） |
| `{{COMMENT}}` | 担当者コメント |
| `{{CUSTOMER_VOICE}}` | お客様の声（アンケートから） |

ヘッダー・フッター・フローティングCTAは `unified-header-footer` から取得。`{{PATH}}` = `../../../`。

出力先: `{area}/works/{NNN}/index.html`

## アンチパターン（絶対に避ける）

| やってはいけないこと | 起きる問題 |
|---|---|
| 写真の中身を確認せずファイル名やEXIF順序だけでキャプション付与 | 中身とミスマッチ |
| Before/After のペアが「同じ場所じゃない」 | Before=分電盤 + After=外観で混乱 |
| 該当ない工程に無理に他の写真を当てはめる | 信頼を失う |
| 画像の圧縮率がページ内で揃わない | 画質ムラ |
| 縦写真と横写真をそのまま横並び | デザイン崩壊 |
| 同じ被写体の写真を複数の場所で使う | 外観をFV・完成写真・関連事例で3回使うなど |

## 必須ルール

1. **Phase 1 で必ず画像前処理スクリプト実行**: 後で気づくと修正が大変
2. **Phase 1.5 の担当者ヒアリングは絶対にスキップしない**: ページの独自性とSEO評価の核心
3. **キャプションは写真の中身に正直に**
4. **Before/After は同じ場所**
5. **完成写真は4枚**（多すぎず少なすぎず）
6. **CSSは `style="aspect-ratio:1/1;object-fit:cover;"` で揃える**
7. **担当者ヒアリングの内容はそのままコピペしない**: 文体を整え、専門用語に補足、読みやすく整形

---

## 担当者ヒアリングが必須な理由（SEO観点）

施工事例ページは「ドアウェイページ」と判定されるリスクが常にあります。テンプレ通りの構成+一般的な工程説明だけでは「どこも同じ」になり、Google のクローラに「価値の低い量産ページ」と見なされ、検索順位が上がりません。

担当者の声は:

- **完全な独自コンテンツ**（他社のサイトには絶対にない）
- **ユーザーの信頼を獲得**（数字や写真より「人の語り」が決め手になる）
- **長期的な検索評価**（E-E-A-T の Experience = 経験を直接的に証明）
- **SNS / 口コミでシェアされやすい**（共感を呼ぶ）

したがって、Phase 1.5 は**スキルの中で最も時間をかけて良いフェーズ**です。
ヒアリングが薄ければページの価値が薄くなる、それくらいの位置付けで扱ってください。
