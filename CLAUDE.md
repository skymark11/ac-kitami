# ac-kitami.com サイト固有ルール

このファイルは **ac-kitami.com（エアコンラボ北見店）** 専用のルールです。
他サイトには適用しないでください。

---

## モバイル起動プロトコル（スマホ版Claudeはまずこれを実行）

### 発動条件

ユーザーが新しい会話の最初に以下のいずれかを発言したとき:

- 「外出先で作業したい」
- 「外出先から進めたい」
- 「スマホから作業始めたい」
- 「これから色々修正したい」
- 「何から始めればいい？」

### 動作

コードに触れる前に、必ず以下のテンプレートをそのまま返答する:

```
おかえりなさい。外出先からの作業開始ですね。

【現在の作業対象】
リポジトリ: skymark11/ac-kitami（エアコンラボ北見店サイト）
※ 別のサイトを触りたい場合は、画面下のリポジトリチップを変更してください。

【進め方の確認】
1. やりたいことを一言で教えてください
   例:「ファーストビューのキャッチコピーを変えたい」
   例:「施工事例を1件追加したい」

2. 自動でやること（指示不要）
   ・ main ブランチは触りません（自動で別ブランチが作られます）
   ・ 作業結果は自動で GitHub に push されます
   ・ 「ブランチ切って」「pushして」と言わなくてOK
   ・ 新規ページ作成時はGoogle Analyticsタグの挿入を忘れない（下記参照）

3. 作業ごとにNOW.mdを自動で更新します（指示不要）
   → 帰宅後にPCで続きをやるとき、NOW.mdを見れば状況がわかります

それでは、何を修正しますか？
```

### 作業終了時のルール（A案: 自動更新）

**1コミットでも作業が完了したら、ユーザー確認を取らず自動でNOW.mdを更新する。**

理由: 外出先では「ここで作業を終える」と意識せず会話を閉じることが多いため、毎コミットごとにNOW.mdを更新しておく。PC帰宅時にNOW.mdを見るだけで状況把握できる状態を維持するのが目的。

NOW.md の「セッション引き継ぎ」セクションに以下フォーマットで追記し、変更内容と同じコミット（または直後の追加コミット）に含めてpush:

```markdown
### YYYY-MM-DD スマホ作業（外出先）

- ブランチ名: claude/xxx-xxx
- 作業内容: （具体的に何をやったか）
- 動作確認: 未実施（パソコンで要確認）
- 次にやるべきこと: （PC帰宅後の継続作業）
- 注意事項: （あれば）
```

## ブランチ運用

- main: GitHub上の正本（GitHub Pagesでデプロイ）
- スマホ版作業: 自動で `claude/xxx-xxx` ブランチが作られる
- PC帰宅後、動作確認OKならmainにマージ → 自動デプロイ

---

## Dropbox同期に関する注意（PC版のみ）

PC版で作業する場合、Dropboxの同期を一時停止してから始めること。
理由: `.tmp` ファイル大量生成、`.git` 競合コピー発生のリスクあり。

スマホ版は関係なし（クラウド上で動くため）。

## Google Analytics

**全ページの `<head>` 内に以下のコードを挿入すること（`<meta charset="UTF-8">` の直後）。**

```html
<!-- Google tag (gtag.js) - 本番環境のみ計測 -->
<script>
    if (location.hostname === 'ac-kitami.com') {
        var script = document.createElement('script');
        script.async = true;
        script.src = 'https://www.googletagmanager.com/gtag/js?id=G-0VXBBQV2PW';
        document.head.appendChild(script);

        window.dataLayer = window.dataLayer || [];
        function gtag(){dataLayer.push(arguments);}
        gtag('js', new Date());
        gtag('config', 'G-0VXBBQV2PW');
    }
</script>
```

**注意事項:**
- 測定ID: `G-0VXBBQV2PW`（変更禁止）
- `location.hostname` チェックにより、ローカル開発環境では計測されない
- 新規ページ作成時に入れ忘れないこと（施工事例詳細ページ、エリアページ等も含む）
